### 3.4 修复明细

#### Fix 1: `server/internal/repo/channel.go::NextMessageSeq`

C018 把 seq 分配迁到 PG sequence (`SELECT nextval('"channel_msg_seq_X"')`)，但漏了 **mirror 回 `channels.seq` 列**。导致：

- `MessageService.MarkRead` 读 `ch.Seq=0` → 写 `channel_members.last_read_seq=0`
- `ListByUserWithPreview` 用 `c.seq - cm.last_read_seq` 算 unread → 永远 0 → 红点永不显示

修复：

```go
// NextMessageSeq 内补上同事务 UPDATE
if err := r.dbOr(ctx, tx).Exec(
    `UPDATE channels SET seq = GREATEST(seq, ?) WHERE id = ?`,
    seq, channelID,
).Error; err != nil {
    return 0, fmt.Errorf("mirror channels.seq: %w", err)
}
```

`GREATEST` 守卫并发 Send 路径乱序 UPDATE。

#### Fix 2-3: 集成测试断言更新

- `m4_batch_e_test.go::TestM4Sync_HappyPath` → 改成断言 `server_event_seq` + `kind.type="events"` + `messages` 是 object
- `m4_message_sync_test.go::TestM4MessageSendThenSync` → 同上 + cursor 字段从 `seq` 改成 `event_seq`
