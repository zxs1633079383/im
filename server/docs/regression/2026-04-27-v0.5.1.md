## 关键观察

1. **cookie-only 鉴权 (5b) 实测生效**：Redis 取真实 mm cookieId `694df75187f3320a1980fe9e` → 服务端 lazy-upsert shadow user (id=4, mm_user_id 反向映射成功)，证明 commit `367f7b8` 的 JWTOrCookie 双栈和 lazy-upsert 路径在 pre 集群跑通。
2. **JWT 优先级正确 (5c)**：同时传 JWT 和 cookie，返回 JWT 对应 user，符合"JWT 优先 + cookie 回落"的设计意图。
3. **OTel metrics 上报失败属环境配置问题**，非代码缺陷：jaeger-cses collector 只暴露 traces 端点，metrics 走 Prometheus pull 模式，可考虑禁用 OTel SDK metrics exporter 避免噪音 INFO 日志。
4. **覆盖率拉低主要在 `internal/repo` (2.1%)**：仓储层本就靠集成测试覆盖，单元测覆盖率低不代表风险。
5. **`scripts/e2e-teardown.sh` 自身有 bug**（exit=1）：Pulsar `topics list` 在没 topic 时返回非零，被 `set -e` 抓到，但前面 PG/Redis 清理已完成，不影响 e2e。建议 teardown 末尾给 Pulsar 段加 `|| true`。
